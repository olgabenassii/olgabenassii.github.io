The first thing we did was define a specific area of knowledge we were particularly interested in;

in this case, we opted for Archaeological sites.


With our first query, we found all the Archaeological Parks present on ArCo, under the class "Cultural Property":



PREFIX rdf: http://www.w3.org/1999/02/22-rdf-syntax-ns#
PREFIX rdfs: http://www.w3.org/2000/01/rdf-schema#
PREFIX arco: https://w3id.org/arco/ontology/arco/

SELECT DISTINCT ?cp ?label ?depiction
WHERE
{?cp rdfs:label ?label ;
rdf:type arco:CulturalProperty .
FILTER(REGEX(?label , "parco archeologico" , "i"))
OPTIONAL {?cp foaf:depiction ?depiction}
}
ORDER BY (?label)


We found out that these results were very detailed, but by running the following query we found out that there were only 15 Archaeological Parks:

ChatGPT also proposed to insert information about the conservation status of the archeological park under examination and about the latest interventions. While the LLMs were capable of providing some general knowledge, we are aware that field specialists should take care of such important data.
For this reason, we decided to propose only the labels that could be used to include such information for every insediamento:

arco-dd:proposedIntervention ;
arco-dd:hasConservationStatus.

These labels already exist in the ArCo ontology, as shown by this more general query that we executed:

PREFIX arco-dd: https://w3id.org/arco/ontology/denotative-description/
PREFIX rdfs: http://www.w3.org/2000/01/rdf-schema#
SELECT*
WHERE
{
?cp rdfs:label ?label ;
arco-dd:proposedIntervention ?proposedIntervention ;
arco-dd:hasConservationStatus ?conservation .
}
LIMIT 100
